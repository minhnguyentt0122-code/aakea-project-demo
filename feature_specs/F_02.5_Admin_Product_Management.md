## M02 — Product Catalogue

---

### F-02.5 Admin Product Management

**Summary**

Admin-Content users create, update, publish, archive, and manage inventory for all products, configurations, and designer profiles in the HomeStyle catalogue.

**User Story**

As an Admin-Content user, I want to create and publish products with all configuration options, material swatches, specification PDFs, and lead times so that the catalogue accurately represents our premium offering and customers can make informed purchase decisions.

**Flow — Happy Path**

1. Admin navigates to `/admin/products/new`.
2. Admin fills in product details: name, description, designer credit, collection(s), materials, dimensions (mm + inches), weight_kg.
3. Admin uploads images (up to 12, JPEG/PNG/WEBP, max 10MB each). S3 pre-signed upload URLs generated per image.
4. Admin uploads material/finish swatch images per configuration option.
5. Admin uploads specification PDF (links to presigned download in F-02.3).
6. Admin adds configuration options (material, finish, fabric grade, base type, size) with per-configuration SKU, price per currency, stock_qty, and lead_time_weeks.
7. Admin clicks 'Publish' → product `status='active'`. Search index updated within 60 seconds.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Admin publishes product with no configurations | Allowed. Product treated as single-configuration. |
| Stock adjustment takes quantity below 0 | 422: 'Stock quantity cannot be negative.' |
| Admin archives product with active orders | Product archived (no new purchases). Existing confirmed/processing orders unaffected. |
| Duplicate SKU | 422: 'This SKU already exists.' |
| Lead time not set for a configuration | Configuration marked as 'Contact for lead time' on storefront until admin updates. |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Products have statuses: draft, active, archived. Only active products visible to customers. |
| BR-02 | Product configurations (`product_configurations` table) store: SKU, price (per currency), stock_qty, lead_time_weeks, weight_kg, and attribute values (material, finish, fabric_grade, base_type, size). |
| BR-03 | Stock adjustments via PATCH are logged in `stock_adjustment_log` with admin ID, reason, before/after quantity. |
| BR-04 | Image uploads use S3 presigned PUT URLs. Images processed (resized/optimised) by Lambda on S3 event. |
| BR-05 | Specification PDFs stored in S3 `product-assets` bucket. Two versions: consumer (standard) and trade (full-resolution with technical drawings). |
| BR-06 | Archiving a product sets `is_archived=true` and removes it from search index within 60 seconds. Product page 301-redirects to parent collection. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| name | String | Yes | Min 3, max 200 chars. | 'Product name is required' |
| description | String | Yes | Min 20, max 5000 chars. HTML allowed (sanitised). | 'Description must be between 20 and 5000 characters' |
| designerId | UUID | No | Must be a valid designer record if supplied. | 'Please select a valid designer' |
| price | Decimal | Yes (per config) | > 0. Max 2 decimal places. | 'Please enter a valid price' |
| sku | String | Yes (per config) | Alphanumeric + hyphens. Max 50 chars. Must be unique. | 'This SKU already exists' |
| stock_qty | Integer | Yes (per config) | ≥ 0. Max 99999. | 'Stock quantity must be 0 or greater' |
| lead_time_weeks | String | No (per config) | Format: 'X–Y weeks' or 'X weeks'. Max 50 chars. | 'Please enter a valid lead time' |
| weight_kg | Decimal | Yes | > 0. Required for white-glove delivery eligibility. | 'Weight is required for delivery calculation' |
