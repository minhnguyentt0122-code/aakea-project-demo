## M03 — Cart & Wishlist

---

### F-03.3 Wishlist Management (including Trade Project Wishlists)

**Summary**

Registered customers save products to a personal wishlist. Verified trade accounts can manage multiple named project wishlists — one per client project — shareable with clients and convertible to cart. TC-28: back-in-stock email sent within 1 hour when a wishlisted item restocks.

**Background / Context**

For consumer buyers, the wishlist captures purchase intent from undecided customers and enables return visits. For trade buyers — interior architects specifying across multiple concurrent client projects — named project wishlists are a core workflow tool: they build a specification list per project, share it with their client for approval, then convert to a purchase order.

**User Story**

As a registered customer, I want to save pieces I like to a wishlist so I can return to them later. As a verified trade account, I want to create named project wishlists per client so I can manage specifications for multiple projects and share them with clients for approval.

**Flow — Happy Path**

1. Customer clicks heart icon on product listing or detail → item added to wishlist (login required).
2. Customer views `/account/wishlist` — all saved items with images, full configuration details, prices (trade pricing for trade accounts), lead times.
3. 'Move to Cart' → adds to cart and removes from wishlist atomically.
4. Trade account: 'New Project Wishlist' → creates named wishlist (e.g. 'Keane Residence — Living Room').
5. 'Share Wishlist' → generates unique shareable URL `/wishlist/[shareToken]` — readable by non-logged-in clients.
6. TC-28: When wishlisted item restocks: customer/trade account receives email within 1 hour (BullMQ hourly job).

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Unauthenticated click of heart icon | Login modal. After login: product auto-added via URL param intent. |
| Wishlisted item goes out of stock | Shown as 'Out of Stock' on wishlist. TC-28: automatically on notify list. |
| Shared wishlist — viewer adds to their own cart | Viewer must log in first. Shared view is read-only. |
| Adding same configuration to wishlist twice | Silent de-duplication — second add ignored. |
| Trade account creates more than 20 project wishlists | 422: 'Maximum 20 active project wishlists per trade account.' |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Wishlist is for registered customers only. No guest wishlist. |
| BR-02 | TC-28: Back-in-stock email sent within 1 hour of restock (BullMQ hourly job). |
| BR-03 | Trade accounts can have multiple named project wishlists (max 20). Consumer accounts: 1 default wishlist. |
| BR-04 | Shared wishlist URL based on a random UUID. Not time-limited. Regenerating issues a new token and immediately invalidates the old URL. |
| BR-05 | Move-to-cart is atomic: item added to cart AND removed from wishlist in a single DB transaction. |
