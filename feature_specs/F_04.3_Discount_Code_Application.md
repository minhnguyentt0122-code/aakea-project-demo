## M04 — Checkout & Payment

---

### F-04.3 Discount Code Application

**Summary**

Customers enter a discount code during checkout. Code is validated at entry (UX feedback) and re-validated server-side at PaymentIntent creation (authoritative check). Trade-only codes are supported for exclusive trade promotions.

**Background / Context**

Discount codes drive seasonal campaigns and customer acquisition for HomeStyle. The double-validation pattern prevents codes from being exploited by applying them to orders where they've since expired or hit usage limits. Trade-only codes enable exclusive promotions for verified trade accounts.

**User Story**

As a customer with a discount code, I want to apply it during checkout and see the discount reflected in my total before I pay.

**Flow — Happy Path**

1. Customer enters discount code in checkout summary field.
2. Frontend calls validation endpoint (optimistic feedback).
3. Server checks: code exists, active, not expired, usage limit not reached, minimum order met, trade-only flag matches account type.
4. Discount shown in order summary as line item: 'Discount (CODE): -$XX'.
5. At PaymentIntent creation: code re-validated server-side. If now invalid: error shown, order total reverted.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Code expired between entry and payment | Caught at re-validation. 'This code is no longer available.' Order total reverts. |
| Trade-only code used by consumer account | 422: 'This discount code is available to verified trade accounts only.' |
| Minimum order value not met | 'This code requires a minimum order of $[X].' |
| OD-04: stacking — two codes attempted | If Option A (one code per order): 'Only one discount code can be applied per order.' |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Code validation performed TWICE: (1) on entry (UX feedback), (2) at PaymentIntent creation (authoritative). |
| BR-02 | OD-04: Stacking rules pending client decision. Default: one code per order. |
| BR-03 | Single-use codes consumed only when order is confirmed (webhook received) — not on checkout entry. |
| BR-04 | Trade-only codes: validated against account `is_trade_verified` flag. Consumer accounts cannot apply trade codes. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| discountCode | String | No | Alphanumeric + hyphens. Max 50 chars. Case-insensitive. Trimmed. | 'This discount code is invalid' / 'This code has expired' / 'Trade accounts only' |
