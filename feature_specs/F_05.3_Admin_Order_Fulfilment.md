## M05 — Order Management

---

### F-05.3 Admin Order Fulfilment

**Summary**

Ops admins view the fulfilment queue, update order status, enter tracking numbers, coordinate white-glove delivery partners, and handle fulfilment exceptions. Made-to-order items require supplier coordination tracked within the order record.

**User Story**

As an Ops admin, I want to manage the order fulfilment queue — confirm orders, enter tracking numbers, coordinate white-glove delivery, and flag exceptions — so that orders are fulfilled accurately within SLA.

**Flow — Happy Path**

1. Admin views `/admin/orders` — filterable by status, date, delivery type (standard/white-glove), made-to-order flag.
2. Admin selects confirmed order → changes status to 'processing'.
3. For made-to-order: admin records supplier order reference and estimated production completion date.
4. Admin enters carrier name and tracking number → status updated to 'dispatched'. Dispatch email triggered.
5. For white-glove: admin notifies delivery partner and records appointment window.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Admin attempts invalid status transition | Status machine rejects (F-05.4). 422 returned. |
| Made-to-order item: supplier delays production | Admin can update estimated dispatch date. Customer NOT automatically notified — admin decides based on extent of delay. |
| TC-17 SLA breached | Order flagged 'Overdue' with red badge. Super Admin receives email escalation. |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Status transitions governed exclusively by F-05.4 Status Machine. |
| BR-02 | TC-17: 4-hour SLA to confirm order from placement. TC-18: 48-hour SLA to dispatch in-stock items from confirmation. |
| BR-03 | Made-to-order: TC-18 dispatch SLA starts from confirmed production completion date, not order placement. |
| BR-04 | All admin order actions written to audit log. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| status | Enum | Yes | Must be valid next status per F-05.4. | 422: 'Invalid status transition' |
| trackingNumber | String | Yes (for dispatch) | Alphanumeric + hyphens. Max 50 chars. | 'Tracking number is required when dispatching' |
| carrier | String | Yes (for dispatch) | Must match configured carrier slug. | 'Please select a valid carrier' |
