## M05 — Order Management

Covers the full lifecycle of a placed order: confirmation, history, tracking, and admin fulfilment. Orders are created exclusively via the Stripe webhook (F-04.2) and progress through a defined status machine.

---

### F-05.1 Order Confirmation & History

**Summary**

Customers receive an order confirmation email immediately after payment, and can view their full order history and order details — including configuration detail and confirmed lead times — from their account.

**User Story**

As a customer who just paid, I want to see a confirmation page and receive a confirmation email with my order details and confirmed lead times so that I know exactly what I ordered and when to expect delivery.

**Flow — Happy Path**

1. Stripe webhook (`payment_intent.succeeded`) received → order record created with `status='confirmed'`.
2. Customer redirected to `/order-confirmation/:orderId` with full order summary including configuration details and lead times per item.
3. Confirmation email (E-03) dispatched via BullMQ immediate job. TC-29 retry policy applies.
4. Customer visits `/account/orders` to view full order history — paginated, most recent first.
5. Customer clicks an order → `/account/orders/:orderId` shows itemised receipt with configuration detail, status, tracking link (when dispatched).

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Customer refreshes /order-confirmation after session cleared | Order found by orderId in URL if customer is authenticated and owns the order. |
| Guest order (OD-01) | Confirmation page accessible via one-time URL. Email sent. |
| Webhook delayed — order not yet created | Page polls GET /orders/:orderId up to 30 seconds. If still missing: 'Your order is being processed — check your email for confirmation.' |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Order history: all orders for authenticated customer, paginated (default 20/page), sorted by `created_at DESC`. |
| BR-02 | Order detail includes: line items (product name, configuration summary, qty, unit price, line total), lead time per line, subtotal, trade discount, shipping, tax, total, delivery address, status, tracking number. |
| BR-03 | TC-17: Confirmation email dispatched as immediate BullMQ job. Retry: TC-29 (3× at 1 min, 1× at 15 min). |
| BR-04 | Order reference format: `HS-[YEAR]-[6-digit sequential ID]` (e.g. `HS-2026-004821`). |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
|---|---|---|---|---|
| **GET** | /api/v1/orders | Order history | `?page=&limit=&status=` | `200: { orders[], total, page }` |
