## M02 — Product Catalogue

Covers all customer-facing product discovery — browsing, searching, viewing details — and admin catalogue management. The primary conversion driver: customers who find the right piece with complete information will buy.

---

### F-02.1 Browse by Room & Category

**Summary**

Customers navigate the product catalogue via collection-based navigation — by room, by designer, by material, and by style — in a hierarchical structure with SSR product listing pages.

**Background / Context**

Premium design buyers shop by inspiration and intent — *'I need a statement piece for my living room'* or *'show me Hans Wegner chairs'* — not by SKU or technical category. Collection-based navigation mirrors how design-led customers think. SSR ensures every category and designer page is SEO-indexed from day one.

**User Story**

As a customer furnishing my home or specifying for a project, I want to browse by room, designer, or material so that I discover relevant pieces in context without having to search by product name.

**Flow — Happy Path**

1. Customer sees collection-based navigation in main header: Living Room, Bedroom, Dining Room, Home Office, Outdoor, By Designer, By Material.
2. Hovering/clicking a collection opens a mega-menu showing subcategories and featured pieces.
3. Customer selects subcategory → navigated to `/collection/[slug]` listing page.
4. Listing page shows: product grid (high-resolution image, product name, designer credit, price, rating), active filter panel, sort controls.
5. Next.js SSR renders the listing page — fully crawlable by search engines.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Collection has no active products | Collection hidden from navigation automatically. URL returns 404 if accessed directly. |
| Customer navigates to archived/deleted collection URL | Return 301 redirect to parent collection page. |
| Filter combination returns zero results | Show 'No products match your filters.' with 'Clear all filters' button. |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Collection hierarchy: Room/Theme → Subcategory → (optional) Designer Collection. Maximum 3 levels. Products can belong to multiple collections (e.g. 'Living Room' AND 'By Designer > Eero Saarinen'). |
| BR-02 | Collections with zero active products are hidden from navigation automatically. |
| BR-03 | Listing pages are SSR in Next.js. Meta title: '[Collection Name] — HomeStyle Premium Design Furniture'. Meta description: configurable per collection in admin. |
| BR-04 | Default sort: 'Most Popular' (by order_count in last 90 days). |
| BR-05 | Price shown in customer's detected/selected currency. EU: VAT-inclusive. US: base price (tax added at checkout). Trade accounts see trade pricing post-login. |
| BR-06 | Archived products (`is_archived=true`) excluded from all listing pages and collection pages. |
| BR-07 | Designer credit displayed on product cards — differentiates premium positioning from generic furniture retailers. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| slug (URL param) | String | Yes (routing) | Alphanumeric, hyphens only. Max 100 characters. Non-existent slug → 404. | 404 page |
| price_min / price_max (query params) | Number | No | Must be positive. price_min ≤ price_max. Invalid values silently ignored. | Silently ignored |
| sort (query param) | Enum | No | Accepted: popular, newest, price_asc, price_desc, rating. Unknown → popular. | Silently defaults to popular |

**UI / Display Rules**

| Condition / State | UI Component | Display Text | Colour / Style | Available Action |
|---|---|---|---|---|
| Collection has products | Product grid | [Product name], [Designer credit], [Price], [Rating stars] | Premium card grid with generous whitespace | Click card → product detail page |
| Zero products after filtering | Empty state | 'No products match your filters.' | Neutral — minimal illustration | 'Clear all filters' button |
| Low stock | Amber badge overlay | 'Low Stock' | Amber (#EF9F27) badge | Still purchasable |
| Out of stock | Red badge overlay | 'Out of Stock' | Red (#E24B4A) badge | Product card still clickable |
| Trade price available | Price display | 'Trade: [price]' visible below RRP for logged-in trade accounts | Subtle secondary pricing display | — |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
|---|---|---|---|---|
| **GET** | /api/v1/categories | Get full collection tree | — | `200: [{ id, name, slug, subcategories[] }]` |
| **GET** | /api/v1/categories/:slug/products | Products with filters | `?sort=&price_min=&price_max=&material=&finish=&designer=&page=&limit=` | `200: { products[], total, page, filters }` |
