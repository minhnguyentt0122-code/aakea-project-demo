## M06 — Returns & Refunds

Handles the post-purchase return lifecycle: customer request, admin review, refund processing via Stripe, and status communication. Made-to-order return eligibility subject to OD-08.

---

### F-06.1 Return Request

**Summary**

Customers initiate a return for a delivered order item within the TC-19 return window, selecting a reason and uploading optional photo evidence. Made-to-order and bespoke items have separate eligibility rules subject to OD-08.

**User Story**

As a customer with a delivered order, I want to request a return online so that I can resolve any issues without calling support.

**Flow — Happy Path**

1. Customer visits `/account/orders/:orderId` and clicks 'Return Item' (visible only for `status=delivered`, within TC-19 window, and if item is eligible per OD-08 policy).
2. Customer selects items to return, selects reason from dropdown, optionally uploads photos.
3. `POST /orders/:orderId/returns` creates return request with `status='return_requested'`.
4. Order status updated to 'return_requested' (F-05.4 transition).
5. Customer receives return confirmation email (E-07). Admin — Trade/Ops notified.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| TC-19: Return window expired | 'Return Item' button hidden. POST returns 422: 'Return window has expired.' |
| Made-to-order item (OD-08) | Return button and eligibility based on OD-08 policy (default: exchange/store credit only unless faulty). |
| Customer tries to return already-returned item | 422: 'A return request already exists for this item.' |
| Photo too large | 422: 'Image must be under 5MB.' |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | TC-19: Return window is admin-configurable (default: 30 days from delivery). Server validates at both UI render AND submission. |
| BR-02 | Made-to-order / custom configuration items: eligibility per OD-08. Default: exchange or store credit only unless item is faulty or does not match specification. |
| BR-03 | Photo evidence optional. Max 3 images, max 5MB each, JPEG/PNG/WEBP. Stored in S3. |
| BR-04 | TC-21: Admin must review return request within 2 business days. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| lineItemIds | UUID[] | Yes | At least 1 item. All must belong to orderId. All must be in returnable state per OD-08. | 'Please select at least one eligible item to return' |
| reason | Enum | Yes | One of: damaged, wrong_item, does_not_match_spec, changed_mind, late_delivery, other. | 'Please select a return reason' |
| notes | String | No | Max 1000 chars. Plain text. | 'Notes must be 1000 characters or fewer' |
| photos | File[] | No | Max 3 files. Max 5MB each. JPEG, PNG, WEBP only. | 'Each image must be under 5MB' |
