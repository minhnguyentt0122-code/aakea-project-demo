## M06 — Returns & Refunds

---

### F-06.3 Refund Processing

**Summary**

Approved returns trigger a Stripe refund (full or partial, cash or store credit). Refund status tracked via Stripe webhooks. TC-30: alert if webhook silent for 24 hours.

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | TC-22: Refund processed within 1 business day of admin marking 'Item Received'. |
| BR-02 | TC-23: Total resolution target ≤12 business days (2 review + 1 refund initiation + up to 10 Stripe processing). |
| BR-03 | Partial refunds permitted for subset of returned items. `refundAmount` set by admin during review. |
| BR-04 | Refund initiated via Stripe Refunds API using original PaymentIntent ID. |
| BR-05 | On `refund.succeeded` webhook: order status set to 'refunded'. Refund confirmation email sent. |
| BR-06 | TC-30: If `refund.succeeded` webhook not received within 24 hours of refund initiation: admin alert triggered. |

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Trade | Ops | Content | Super Admin |
|---|---|---|---|---|---|---|---|---|
| **POST** | /api/v1/admin/returns/:returnId/refund | Initiate Stripe refund | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| **POST** | /api/v1/webhooks/stripe | Stripe refund webhook | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
|---|---|---|---|---|
| **POST** | /api/v1/admin/returns/:returnId/refund | Initiate refund | `{ refundAmount }` | `200: { refundId, status: 'refund_processing' }` |

**TC References:** TC-22 (1 business day processing), TC-23 (≤12 business day total resolution), TC-30 (24h webhook alert)
