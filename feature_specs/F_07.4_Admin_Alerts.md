### F-07.4 Admin Alerts

**Summary**

Super Admins and Ops admins receive automated alerts for critical events: payment failures, webhook errors, SLA breaches, email dead-letter entries, stock depletion, and overdue trade account applications.

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Trade | Ops | Content | Super Admin |
|---|---|---|---|---|---|---|---|---|
| **GET** | /api/v1/admin/alerts | List system alerts | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/alerts/:alertId | Acknowledge alert | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/alerts/settings | Update alert preferences | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
|---|---|---|---|---|
| **GET** | /api/v1/admin/alerts | List alerts | `?acknowledged=false&severity=critical\|warning&page=` | `200: { alerts[] }` |
| **PATCH** | /api/v1/admin/alerts/:alertId | Acknowledge | `{ acknowledged: true }` | `200: { alert }` |
| **PATCH** | /api/v1/admin/alerts/settings | Update preferences | `{ emailRecipients[], slackWebhook?, thresholds{} }` | `200: { settings }` |

**TC References:** TC-27 (low-stock), TC-28 (back-in-stock), TC-29 (email failure), TC-TRA-01 (trade application SLA)

**Acceptance Criteria**

| # | Criteria | Test Type | Status |
|---|---|---|---|

| AC-02 | Overdue trade application (TC-TRA-01) triggers Super Admin alert. | Integration | — |
| AC-03 | Acknowledged alerts hidden from default view. | Unit | — |
