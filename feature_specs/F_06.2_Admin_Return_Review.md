## M06 — Returns & Refunds

---

### F-06.2 Admin Return Review

**Summary**

Ops admins review submitted return requests within TC-21: 2-business-day SLA, approve or reject them with reasoning, and either issue a return label/schedule white-glove pickup or reject with explanation.

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Trade | Ops | Content | Super Admin |
|---|---|---|---|---|---|---|---|---|
| **GET** | /api/v1/admin/returns | List return requests | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| **GET** | /api/v1/admin/returns/:returnId | Return detail | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/returns/:returnId | Approve or reject | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | TC-21: Admin must review within 2 business days. SLA breach flags item in admin panel and alerts Super Admin. |
| BR-02 | Approval decision: 'cash_refund' (standard items unassembled), 'exchange_or_credit' (assembled or made-to-order per OD-08), 'replacement' (damaged on arrival). |
| BR-03 | For white-glove returns: admin can schedule white-glove pickup from customer's address rather than requiring customer to ship. |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
|---|---|---|---|---|
| **GET** | /api/v1/admin/returns | List returns | `?status=&dateFrom=&page=` | `200: { returns[], total }` |
| **PATCH** | /api/v1/admin/returns/:returnId | Approve / reject | `{ decision: 'approved'\|'rejected', refundType?, refundAmount?, returnMethod?: 'customer_ships'\|'white_glove_pickup', adminNotes? }` | `200: { return }` |

**TC References:** TC-21 (2 business day review SLA)

**Acceptance Criteria**

| # | Criteria | Test Type | Status |
|---|---|---|---|
| AC-01 | PATCH with `decision='approved'` triggers refund processing (F-06.3). | Integration | — |
| AC-02 | PATCH with `decision='rejected'` updates status and sends rejection email with reason. | Integration | — |
| AC-03 | TC-21 SLA breach triggers Super Admin email alert. | Integration | — |
