### F-02.4 Delivery Availability Checker

**Summary**

Customers enter ZIP/postal code on the product detail page to check delivery availability, estimated dates, delivery method options, and white-glove service eligibility before committing to checkout.

**Background / Context**

For large, high-value furniture pieces — sofas, beds, wardrobes — delivery logistics are a major purchase consideration. Learning at checkout that white-glove delivery isn't available, or that delivery to your area takes 6 weeks, after spending time in checkout is a top abandonment trigger. The checker surfaces this information at the product evaluation stage, before cart commitment.

**User Story**

As a customer considering a large furniture piece, I want to check delivery options and white-glove availability for my address before adding to cart so that I can make an informed purchase decision.

**Flow — Happy Path**

1. Customer enters ZIP/postal code in delivery checker on product detail page.
2. Server checks against `serviceable_areas` table.
3. If serviceable: shows available delivery methods (standard parcel, threshold, white-glove) with estimated dates and prices.
4. If not serviceable: shows unavailability message. Does NOT block cart add — customer may be buying for a different address.
5. Browser caches result for same ZIP+product combination for 1 hour.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Invalid ZIP/postal code format | Inline error: 'Enter a valid ZIP / postal code.' No server call made. |
| Remote area with delivery surcharge | Show standard delivery option with surcharge: 'Standard Delivery to remote area: $[X+surcharge]'. |
| Carrier API timeout (if OD-09 Option B) | Fall back to DB lookup or show: 'Unable to check delivery for this address. You can verify at checkout.' |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | ZIP/postal code checked against `serviceable_areas` table in PostgreSQL. |
| BR-02 | Delivery estimate = today + TC-17 (4-hr fulfillment SLA) + TC-18 (48-hr dispatch SLA) + carrier transit days for destination zone. |
| BR-03 | White-glove delivery shown only if: product `weight_kg` exceeds admin-configured threshold AND destination ZIP is in white-glove service zone (CR-11). |
| BR-04 | Checker result cached in browser `sessionStorage` for 1 hour per ZIP+product combination. |
| BR-05 | Not serviceable ZIP does not block 'Add to Cart' — some customers check for a third-party delivery address. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| zip | String | Yes (for check) | US ZIP: 5 digits or ZIP+4. EU: 4-6 digits. UK: standard postcode regex. Client-side validated before server call. | 'Please enter a valid ZIP / postal code' |

| Condition / State | UI Component | Display Text | Colour / Style | Available Action |

| Delivery available — standard only | Delivery options list | 'Standard Delivery: Est. [date range] — $[X]' | Green checkmark | — |
| White-glove available | Delivery options list | 'Standard: Est. [date] — $[X]' and 'White-Glove: Room of choice + packaging removal: Est. [date] — $[Y]' | Green checkmarks; White-Glove line highlighted | — |

| Invalid ZIP format | Inline field error | 'Please enter a valid ZIP / postal code' | Red border + helper text | Server call blocked |
