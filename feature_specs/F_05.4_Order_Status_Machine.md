## M05 — Order Management

---

### F-05.4 Order Status Machine

**Summary**

Defines and enforces all valid order status transitions. No direct DB writes bypass this machine. Made-to-order items have additional statuses to track production lifecycle.

**Business Rules / Valid Transitions**

| From Status | Permitted Next Statuses | Triggered By |
|---|---|---|
| pending | confirmed, failed | Stripe webhook |
| confirmed | processing, cancelled | Admin (F-05.3) / Customer cancel (F-05.1) |
| processing | in_production, dispatched, cancelled | Admin (F-05.3) |
| in_production | production_complete, cancelled | Admin (made-to-order only) |
| production_complete | dispatched | Admin |
| dispatched | delivered | Admin or carrier webhook |
| delivered | return_requested | Customer (F-06.1) |
| failed | — (terminal) | Stripe webhook |
| cancelled | — (terminal) | Admin / Customer |
| return_requested | return_approved, return_rejected | Admin (F-06.2) |
| return_approved | refund_processing | System (F-06.3) |
| refund_processing | refunded | Stripe webhook |
| refunded | — (terminal) | — |
| return_rejected | — (terminal) | Admin (F-06.2) |
